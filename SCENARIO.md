# PremierLeagueNerd — Database & Application Scenario

PremierLeagueNerd is a Premier League information platform built with Next.js, Express, and PostgreSQL. The database stores all core football data across seven tables — stadiums, teams, players, matches, match events, users, and sessions — each connected through foreign keys. Stadiums are linked to teams, teams are linked to players and matches, and individual goal or assist events are recorded per player per match. User accounts are stored separately with bcrypt-hashed passwords and managed through server-side sessions using HTTP-only cookies.

When a user first visits the platform, they can register or log in. The first account ever created is automatically assigned the admin role, while all subsequent registrations receive the viewer role. Once authenticated, a viewer can browse all Premier League fixtures grouped by date, check the live league standings computed from finished match results, explore player profiles filtered by name or position, and view statistics including the top scorers, top assist providers, and teams with the most clean sheets.

Administrators have additional access to a management panel where they can add new fixtures, update match scores and statuses, and delete players. Player deletion is handled inside a database transaction that first removes all related match events before deleting the player record, ensuring the database stays consistent. Any changes made by an admin are immediately reflected across the standings, statistics, and matches pages for all users.
